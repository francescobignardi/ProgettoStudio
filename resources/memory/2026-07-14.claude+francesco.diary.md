Concetto correlato fissato: `artisan` è un **file PHP fisico** nel filesystem, non un comando globale — infatti l'errore è "Could not open input **file**", non "command not found". `php artisan X` = "PHP, esegui lo script `./artisan` con argomento `X`".

**Esito (dopo output route:list)**
Quattro route base, tutte previste per uno scheletro Laravel appena creato:
- `GET|HEAD /` → `routes/web.php:5` (la landing appena vista dal browser). **Questa è "casa nostra"** — dove aggiungeremo le route custom.
- `GET|HEAD storage/{path}` → serve file caricati dagli utenti (parametro dinamico `{path}` in graffe = segmento URL variabile).
- `PUT storage/{path}` → speculare per upload. Stesso path, verbo diverso = route diverse. Il router discrimina anche per metodo HTTP.
- `GET|HEAD up` → health check di Laravel 11+ (endpoint `/up` che risponde 200 se l'app è viva; usato in prod da load balancer/Kubernetes).

Le tre di sistema stanno in `vendor/laravel/framework/...` — sono "regalate" dal framework, non definite dal codice utente.
