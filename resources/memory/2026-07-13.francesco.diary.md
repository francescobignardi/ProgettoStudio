## 10:02 — Apertura Fase 4: Laravel installato e collegato a MySQL

> Bozza creata da claude. Da rivedere/integrare.

- Scelta iniziale non banale: Sail (scorciatoia) vs Dockerfile custom esteso (strada lunga). Andati sulla seconda per coerenza con quello che ho appena imparato (`pdo_mysql` aggiunto a mano l'8 luglio) e perché sul CV "so scrivere il Dockerfile che serve a Laravel" pesa di più di "so usare Sail".
- Verificate con `php -m` le estensioni già presenti nell'immagine: 14/14 obbligatorie Laravel c'erano già. Mancavano solo `bcmath` e `zip`. Aggiunte con una `RUN` unica al Dockerfile che fa anche `apt-get install libzip-dev` (zip in PHP dipende da una libreria di sistema) e pulizia liste apt. Best practice standard.
- Aggiunto Composer nell'immagine con `COPY --from=composer:2 ...`. Prima volta che vedo il pattern multi-stage: si può "pescare" un binario da un'immagine ufficiale esterna senza reinstallarlo. Elegante.
- Chiarita una confusione mia: `composer` (il programma) ≠ `composer.json` (il file di configurazione del progetto). Il programma sta *nell'immagine*, il file sta nel container tramite bind mount. Sono cose diverse.
- Laravel installato in `backend/` con `composer create-project laravel/laravel backend`. Lo scheletro si è configurato praticamente da solo: chiavi generate, database SQLite creato al volo, tre migrazioni base applicate. Riflessione: quanta developer experience c'è nei framework moderni rispetto al framework custom di lavoro dove ogni progetto si compone a mano copiando da altri.
- Cambiato `backend/.env` per usare **il nostro MySQL** invece di SQLite (che era il default Laravel). Due typo iniziali su nome database e password vuota — errori simili a quelli che avevo già visto con `db-check.php` l'8 luglio, quindi già in dominio.
- Toccato con mano due concetti importanti di Docker sbagliando: (a) `artisan` sta in `/app/backend`, non in `/app` — il flag `-w` di `docker exec` permette di cambiare working dir al volo; (b) **dentro** il container non c'è Docker — è ovvio scritto così, ma quando sei nel prompt ci si scorda. Fuori: `docker exec`. Dentro: comandi nudi.
- `php artisan migrate:fresh` da dentro il container → 3 migrazioni base applicate. Verifica finale con `SHOW TABLES;` sul MySQL del compose: 9 tabelle presenti (users, cache, jobs, sessions, migrations, ecc.). Laravel connesso al MySQL. Fase 4 partita.
